# Project map

## Purpose

Telegram bot for managing a Docker-based VPN server stack. Imported from `mercurykd/vpnbot`.

Supported services in upstream README:

- VLESS / Xray
- NaiveProxy
- OpenConnect
- WireGuard
- Amnezia
- AdGuardHome
- MTProto
- PAC
- automatic SSL

## Entrypoints

- `scripts/init.sh` - original one-command install script.
- `makefile` - operational commands for build/start/stop/logs/backup.
- `docker-compose.yml` - service graph and container config.
- `app/init.php` - PHP bot bootstrap.
- `app/bot.php` - main Telegram bot logic.
- `app/config.php` - local secret config generated by installer, ignored by git.
- `config/hwid.json` - tracked empty default for HWID device storage; runtime may update it.
- `config/wg0.conf`, `config/wg1.conf` - runtime WireGuard config files; startup creates them if missing.

## Common commands

```sh
docker compose config
make u
make r
make d
make ps
make l
```

## Dangerous commands

These mutate or delete local Docker/git state:

```sh
make delete
make reset
make dv
```

Use only after explicit confirmation.

## Agent-orchestrator target

Use this repo path as the project path:

```powershell
D:\project\kkk-go-bot
```

Suggested `ao` registration:

```powershell
ao project add --path "D:\project\kkk-go-bot" --id kkk-go-bot --name kkk-go-bot --worker-agent codex --orchestrator-agent codex
```
